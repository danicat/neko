# Open Bugs & Issues - Neko MCP

**Date:** March 21, 2026

During comprehensive tool testing, the following functional bugs were observed and need to be addressed:

## 1. `rename_symbol` Tool is a No-Op
- **Symptom**: Executing the `rename_symbol` tool reports success (e.g., `✅ Rename Successful`), but reading the file back reveals that the file content was not actually changed.
- **Expected Behavior**: The file(s) should reflect the renamed symbol appropriately across the codebase and flush the changes to disk.

## 2. `find_references` Output Rendering Issue
- **Symptom**: The output for `find_references` is not rendering locations properly. It displays raw structural tags like `(in 'go')` and misclassifies regular source files as `[TESTS]`. It appears to be printing internal struct data instead of formatting a clean, human-readable markdown response.
- **Expected Behavior**: Should cleanly format standard reference locations and accurately group `[SOURCE]` vs `[TESTS]` files.

## 3. Working Directory (CWD) Mismatch for Tools
- **Symptom**: Calling `open_project(dir="path/to/subproject")` establishes an LSP context for that subproject but does not update the underlying working directory context for other tools. Tools like `test_mutations` that accept `dir: "."` end up running against the server's initial root launch directory rather than the target project directory.
- **Expected Behavior**: Context-aware tools should respect the directory of the currently active project context, or `open_project` should properly sync the active directory state.
